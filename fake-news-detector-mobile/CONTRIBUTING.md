# دليل المساهمة

شكراً لاهتمامك بالمساهمة في مشروع **Fake News Detector**! 🎉

## كيفية المساهمة

### 1. البلاغات عن الأخطاء (Bug Reports)

إذا وجدت خطأ، يرجى فتح Issue جديد مع المعلومات التالية:
- وصف واضح للمشكلة
- خطوات إعادة الإنتاج
- السلوك المتوقع والفعلي
- لقطات شاشة إن أمكن
- معلومات الجهاز والنظام

### 2. طلبات الميزات (Feature Requests)

لاقتراح ميزة جديدة:
- اشرح الميزة بوضوح
- اشرح الفائدة والحالات الاستخدام
- قدم أمثلة إن أمكن
- ناقش التنفيذ المحتمل

### 3. Pull Requests

#### خطوات المساهمة:

1. **Fork المستودع**
   ```bash
   git clone https://github.com/youness-bourek/fake-news-detector.git
   cd fake-news-detector-mobile
   ```

2. **أنشئ فرع جديد**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **قم بالتغييرات**
   - اتبع معايير الكود
   - أضف اختبارات إن أمكن
   - حدّث التوثيق

4. **اختبر التغييرات**
   ```bash
   npm run test
   npm run lint
   ```

5. **اكتب رسالة commit واضحة**
   ```bash
   git commit -m "feat: add amazing feature"
   ```

6. **ادفع التغييرات**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **افتح Pull Request**
   - اشرح التغييرات
   - اربط أي Issues ذات صلة
   - أضف لقطات شاشة إن أمكن

## معايير الكود

### أسلوب الكود
- استخدم TypeScript
- اتبع معايير ESLint
- استخدم Prettier للتنسيق
- اكتب كود نظيف وقابل للصيانة

### التعليقات والتوثيق
- أضف تعليقات للأجزاء المعقدة
- وثّق الدوال والمكونات
- استخدم JSDoc للتوثيق

### الاختبارات
- اكتب اختبارات للميزات الجديدة
- تأكد من نجاح جميع الاختبارات
- حافظ على تغطية الاختبارات > 80%

## عملية المراجعة

1. سيتم مراجعة PR الخاص بك
2. قد يطلب المراجعون تعديلات
3. بعد الموافقة، سيتم دمج PR
4. شكراً على مساهمتك! 🎉
